# SingSong Requirements Trace

이 원장은 변경 전 자료, Git 역사, v3.2 요구를 현재 구현과 검증 소유자에 연결한다. 초기 자료의 exact hash/size는 `MATERIAL_INVENTORY.md`, Markdown 완독 근거는 `MD_READ_LEDGER.md`, refs/object/commit 상세는 `GIT_HISTORY_AUDIT.md`가 정본이다. 표의 범위 표기는 양 끝을 포함하며 해당 ID를 하나도 생략하지 않는다.

상태 어휘:

- `IMPLEMENTED`: 현재 코드/설정/문서에 직접 반영됨. 실행 결과는 `VERIFICATION_REPORT.md`가 최종 판정한다.
- `DOC_TEST_OPS`: 구현 판단, 테스트, 디자인 또는 운영 근거로 반영됨.
- `SUPERSEDED`: 최신 계약과 충돌해 구현하지 않되 자료는 보존됨.
- `EXCLUDED`: 구체적 보안/제품/기술 이유로 출하에서 제외됨.
- `PARTIAL_REPO`: 저장소 안에서 추가 구현 또는 자동 증거가 필요함.
- `BLOCKED_EXTERNAL`: 외부 권리/키/인프라/도메인/실기기/사람 권한 없이는 완료할 수 없음.
- `NOT_RUN`: 구현·테스트 소유자는 있으나 최종 run에서 실행 증거가 아직 없음.
