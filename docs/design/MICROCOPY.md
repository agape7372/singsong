## 1. 톤 정의

**확정 톤**: 반말 친구톤. 코인노래방에서 옆에 있는 친구가 툭 던지는 말투. "계산이 완료되었습니다" 류의 시스템 말투는 전 화면 금지.

### 1-1. 톤 스펙트럼 (허용 범위)

| 구역 | 캐주얼 허용 범위 | 이모지 |
|---|---|---|
| **CTA·짧은 상태 문구·토스트** | 반말 + 구어 축약("ㄱㄱ?", "완전", "딱이야") 허용 | 문장당 최대 1개, 문장 끝. 남발 금지 |
| **본문 설명·안내 문구** (검색 placeholder, 빈 상태 설명 등) | 반말이되 축약형("ㄱㄱ", "ㅇㅋ" 등 자음축약)은 지양 — 완전한 구어체 문장 | 선택적 1개, 없어도 무방 |
| **버튼 라벨** | 반말 동사형("저장할래", "부를래" 대신 간결 명령형 "저장하기"류도 허용 — §4 CTA 참고) | 미사용 |
| **예외 구역(§1-2)** | 반말 유지, 축약·이모지 전면 금지 | 미사용 |

- 이모지는 **문장 끝에 최대 1개**, 같은 화면에 이모지 2개 이상 동시 노출 금지(과유불급).
- "ㄱㄱ?" 같은 초성/자음 축약은 **CTA·티켓 카피처럼 캐주얼함이 브랜드 정체성인 지점에만** 허용. 안내문·에러문에는 쓰지 않는다.
- 물결(~), 느낌표 연타(!!) 금지 — 과장된 텐션 대신 담백한 확신 톤.

### 1-2. 예외 구역 (명확성 우선)

아래 3개 상황은 **반말은 유지하되** 구어 축약·이모지를 전면 금지한다. 실수로 데이터를 잃거나 공개 범위를 오해하는 상황은 재미보다 정확도가 우선이다.

1. **데이터 유실 경고** (기기 초기화·앱 삭제 시 리스트 사라짐 안내, persist 실패 고지)
2. **공개범위 고지** (링크 공유 시 "이 링크 아는 사람 누구나 볼 수 있어" 류)
3. **삭제 확인** (리스트·항목 영구 삭제 ConfirmDialog)

이 구역은 §4 표에 `[예외구역]` 표시.

## 3. 표기 규칙

### 3-1. 원화
- **"원" 고정.** `₩` 기호 미사용(모노스페이스 금액 숫자는 디자인 정본 — DESIGN_SYSTEM 참조. 여기서는 텍스트 표기만 규정).
- 천단위 콤마 필수: `3,000원`, `12,000원`. 콤마 없는 `3000원` 금지.
- "1인당" 접두: `1인당 3,000원` (조사 없이 값 앞에 라벨 고정 — §3-5 참고).

### 3-2. 시간(소요시간·이용시간)
| 맥락 | 형식 | 예 |
|---|---|---|
| 계산 결과·스텝퍼·칩·티켓 요약 (컴팩트 수치) | 항상 `N분` (60 넘어도 분 단위 유지, 시간 환산 안 함) | `90분`, `150분` |
| 서술형 안내 문장(온보딩·빈 상태 설명 등 사람이 읽는 문장 속) | 60분 이하는 `N분`, 60분 초과는 `H시간 M분`(0분이면 `H시간`만) | `90분` → 문장 안에서도 `90분` 유지 가능하나, `150분`처럼 큰 값은 `2시간 30분`으로 완화 허용 |
| 개별 곡 재생시간 (SongRow) | `mm:ss` 콜론 표기 (분·초 각 정수, 초는 2자리 0패딩) | `3:45`, `0:58` |

> 원칙: **숫자 카드·계산 결과·리모컨 정서가 있는 곳은 무조건 "N분"** (BUILD_PLAN §6 `duration_sec` 파생값과 1:1 대응, 반올림 규칙은 BUILD_PLAN §6 정본). 문장체 설명문만 가독성을 위해 시:분 병기를 허용한다.

### 3-3. 곡수·인원 단위
- 곡수: `24곡` (조사 없이 숫자+단위 고정). 0곡은 "곡 없음" 대신 §4 빈 상태 문구 사용.
- 인원: `3명`.
- 티켓 좌석번호 모티프(=곡수)는 DESIGN_SYSTEM/COMPONENTS 정본 — 여기서는 텍스트 라벨만.

### 3-4. 날짜
- 히스토리 등 상대적 맥락: `오늘`, `어제`, `3일 전`(2~6일 전), 7일 이상은 `7월 3일`(연도 생략, 당해 연도 기준).
- 연도가 바뀌는 항목(과거 기록)만 `2025년 7월` 형식으로 예외 표기.
- 절대 날짜가 필요한 법적 고지·타임스탬프 텍스트: `YYYY.MM.DD` 형식(예외구역 톤 적용, §1-2).

### 3-5. 조사 처리 (동적 값 뒤 조사 회피)
동적 값(곡 제목·리스트 제목·인원수 등) 뒤에 오는 한국어 조사(을/를, 이/가, 은/는, 로/으로)는 받침 유무에 따라 형태가 달라져 정적 문자열로 안전하게 붙일 수 없다. **회피 문형을 우선 사용**한다.

| ❌ 조사 결합(깨짐 위험) | ✅ 회피 문형 |
|---|---|
| `"{title}을(를) 삭제할까?"` | `"이 리스트, 삭제할까?"` (지시대명사로 대체) |
| `"{song}이(가) 담겼어"` | `"담았어!"` (주어 생략) |
| `"{title}로(으로) 이동합니다"` | `"이동할게"` (목적어 생략, 이동 자체가 문맥상 명확할 때만) |
| `"{name}님의 플리"` | `"{name}의 플리"` (이름+"의"는 받침 무관 안전, "님" 존칭은 반말 톤과 불일치하므로 미사용) |

- **불가피하게 조사가 꼭 필요한 경우**(리스트 상세 제목 등)에만 프로젝트 표준 조사結합 유틸(구현 시 `lib/josa.ts` 등 — 실측 필요, 소유 정본 아님)을 통해 받침 판별 후 동적 결합한다. 이 규칙은 마이크로카피 정본이 아니라 엔지니어링 구현 사항이므로 ARCHITECTURE.md에 위임 — 여기서는 "가능하면 회피 문형"만 규정.

### 3-6. 맞춤법·띄어쓰기
- 표준 맞춤법 준수, 단 구어체 종결어미(`~할래`, `~할까`, `~야`)는 허용.
- 외래어 표기: `플리`(플레이리스트 줄임, 확정 용어), `노래방`, `코인노래방`(상표 아님, 일반명사).
- 숫자+단위 사이 띄어쓰기 없음: `24곡`, `3명`, `90분`(붙여쓰기 고정 — 컴팩트한 리모컨 정서).

## 5. 접근성 문구 (아이콘 단독 버튼 aria-label)

아이콘만 있고 텍스트 라벨이 없는 버튼은 스크린리더 사용자에게 행동을 알려줘야 한다. 반말 톤을 적용하지 않고 **명사형으로 행동을 명시**한다(스크린리더 관례).

| 버튼 | aria-label |
|---|---|
| 닫기(BottomSheet, Toast, 배너) | "닫기" |
| 검색창 지우기(SearchInput clear) | "검색어 지우기" |
| 곡 순서 변경 핸들 | "순서 바꾸기" |
| 곡 삭제(SongRow) | "곡 빼기" |
| 리스트/티켓 삭제 | "삭제하기" |
| PWA 설치 배너 닫기 | "설치 배너 닫기" |
| 공유 시트 닫기 | "공유 닫기" |
| 뒤로가기(라우트 back) | "뒤로 가기" |
| 히스토리 항목 재담기 | "다시 담기" |
| Stepper 증가/감소 | "인원 늘리기" / "인원 줄이기" (대상 명사는 컨텍스트에 맞게 치환) |

## 검증 체크리스트

1. **grep 안티패턴**: 구현 코드에서 `습니다`, `하십시오`, `됩니다`, `해주세요`(존댓말 계열) 문자열 grep — 0건이어야 함(예외구역 포함 전부 반말이므로).
2. **원화 표기 일관성**: 코드베이스에서 `₩` 문자 사용처 grep — 0건.
3. **천단위 콤마**: `toLocaleString('ko-KR')` 또는 동등 포매터를 거치지 않고 금액을 문자열 템플릿에 직접 삽입하는 곳이 있는지 리뷰.
4. **이모지 남발 검사**: 한 문자열 안에 이모지 2개 이상 포함 여부 정규식 검사(`/\p{Emoji}.*\p{Emoji}/u`) — 0건.
5. **예외구역 준수**: `[예외구역]` 표시된 5개 키(list.clear.confirm.*, ticket.public.notice, common.confirmDialog.confirm.delete, common.persist.warning.*)의 실제 렌더 문자열에 이모지·자음축약(`ㄱㄱ`, `ㅇㅋ` 등) 포함 여부 육안 검토.
6. **버튼 라벨 모호성 검사**: "확인"/"취소" 단독 문자열이 실제 버튼 텍스트로 렌더되는 곳이 있는지 코드 검색 — §4/§고찰 원칙 위반이면 수정.
7. **aria-label 커버리지**: §5 목록에 있는 아이콘 전용 버튼 컴포넌트 각각에 `aria-label` prop이 실제로 전달되는지 COMPONENTS.md 구현체 대조.
8. **문구 사전 키 중복/누락**: 실제 구현에서 사용하는 문자열 상수 키 목록을 뽑아 §4 표의 키와 diff — 표에 없는 신규 문구가 생기면 이 문서를 먼저 갱신(README §5 규칙).
