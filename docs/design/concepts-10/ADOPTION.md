# CUTLINE 디자인 채택 기록

> **상태**: 우선 채택  
> **결정일**: 2026-07-21  
> **대상**: SingSong P0 모바일 플랜 흐름

## 결정

10개 탐색안 중 `01 CUTLINE — 에디토리얼 발권지`를 SingSong의 기본 시각 방향으로 채택한다.

첫 구현 슬라이스는 다른 시안의 스타일을 혼합하지 않고 CUTLINE만으로 완성한다. 이후 실제 사용자 검증에서 문제가 확인되면 구조적 장점이 명확한 요소만 별도 결정으로 가져온다.

## 채택 이유

- `곡 담기 → 시간·비용 확인 → 티켓 만들기`가 하나의 연속된 발권지로 읽힌다.
- 제품 정본의 `Session Strip` 개념과 가장 직접적으로 연결된다.
- 장식보다 곡 순서와 계산 근거가 먼저 보여 현장 사용성이 좋다.
- 검색 ledger, 발권 티켓, 공유 화면으로 같은 형태 문법을 확장하기 쉽다.
- 유행성 강한 gradient·glass·과도한 카드 더미 없이도 젊은 인상을 유지한다.

## 잠그는 시각 계약

- **색**: cream `#FAF7F0`, ink `#15131A`, action rose `#FF3D6E`, money `#B76E00`.
- **구조**: 번호가 붙은 연속 ledger와 Queue→Calculator 경계의 절취선 한 번.
- **면**: 화면의 강한 raised surface는 Working Strip 하나만 사용.
- **타입**: 큰 제목은 굵은 산세리프, 숫자·시간·금액은 tabular/mono 계열.
- **행동**: CTA는 pill이 아닌 사각형 10–12px radius, 최소 높이 52px.
- **금지**: gradient, glassmorphism, glow, 종이 텍스처, 3D, 반복되는 독립 카드, 장식용 음표·마이크.

## 구현 순서

1. 모바일 플랜 홈의 Working Strip과 ActionDock
2. 같은 ledger 문법의 검색 화면
3. 계산 결과를 확정하는 TicketCard
4. 같은 정보 위계의 공유 화면
5. wide layout과 dark mapping

## 재검토 게이트

첫 번째 vertical slice에서 다음을 확인한 뒤 세부 조정을 허용한다.

- 처음 보는 사용자가 2분 안에 3곡 이상과 계산 결과에 도달하는가
- 5초 노출 후 곡 수·시간 범위·총액·1인당 금액을 구분해 말할 수 있는가
- 하단 CTA가 콘텐츠를 가리거나 계산 입력과 경쟁하지 않는가
- 작은 화면과 200% 확대에서도 ledger와 금액 범위가 잘리지 않는가

재검토 전까지 나머지 9개 시안은 참고 자료이며 구현 소스가 아니다.
