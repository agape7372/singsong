# Ticket directions — 티켓 방향 탐색

> **용도**: 발권 티켓의 시각 계보를 고르기 위한 내부 탐색물. 프로덕션 bundle·로고·UI asset으로 출하하지 않는다.
> **정본**: [`../VISUAL_MOTION_DIRECTION.md`](../VISUAL_MOTION_DIRECTION.md). 이 폴더의 PNG 안에 있는 글자·숫자·구조보다 정본이 항상 우선한다.
> **원칙 문서**: [`TICKET_PROMPT_PRINCIPLES.md`](./TICKET_PROMPT_PRINCIPLES.md) — 시안1. 이 시안들을 만든 근거와 채점표가 여기 있다.

## 채점 결과 (§16 기준, 100점)

| 항목 | 배점 | 03 | 04 | 05 | 06 |
|---|---:|---:|---:|---:|---:|
| 티켓다움 1차 신호의 정직성 | 15 | 15 | 8 | 12 | 10 |

감점 근거:

- **03** — 4:5 프레임에 `serial`이 없다(정본 §9-3 필수 정보 누락). 세로 1:2라 OG 1200×630에 불리. M형은 미싱이 없어 정본 §2-1 Punch 정의와 충돌(1건).
- **04** — 바코드가 스캔 불가능한 장식이다(§8-3 실패 모드 직격). 한글 라벨까지 monospace라 정본 §4-1 위반. 종이 말림·톱니 가장자리는 §1-4 스큐어모피즘 금지와 긴장하고 CSS 재현도 어렵다. 세로 스트립이라 OG 최악.
- **05** — 미싱이 실제로 있어 Punch 정의와 호환되는 유일한 후보. 팔레트가 정본 §3 밖(네이비/오렌지)이라 충돌 1건. 4:5 프레임에 가로 티켓을 넣으니 여백만 커지고 활자가 작아진다. 뒷면 약관이 깨진 라틴 더미 텍스트(리터럴 미지정 슬롯).
- **06** — 핑크·블루 오버프린트가 육안으로 보라 그라디언트처럼 읽혀 정본 §2-5와 근접 위험. 격자 룰선의 빈 셀이 많아 "정보 없는 장식"이다. 리소 도트 스크린·핀오차는 CSS 재현이 어렵다. 팔레트 전면 이탈.

**동점 처리**: 04와 06이 70점 동점. §16 규칙 (2)에 따라 정직성+slop 합산(04=23, 06=25)으로 06 우선 — 다만 06은 정본 충돌이 가장 크다.

## 복사 금지 요소

- 이미지에 생성된 문구·숫자·가격·시간·버튼 문장. 정본 계약은 자유 티켓 제목 없음 + 시간·금액 **범위 보존**이다.
- 보드 안의 route/tab 수, 별도 계산 화면, 곡 목록 행 수, 공유 버튼 구성.
- raster 조각을 잘라 만든 로고·아이콘·배경.
- 생성 이미지 속 바코드·QR. 실제 코드는 정본 §9-4 규정대로 `publicUrl` 기반 코드 렌더러가 만든다.

실제 제품은 semantic HTML, CSS variables, Base UI primitive, `motion/react`, 사람이 검수한 SVG로 재구성한다.
