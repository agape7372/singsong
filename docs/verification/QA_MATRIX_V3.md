## Severity와 release rule

- Critical: secret/data exposure, destructive loss, public enumeration, wrong money computation, core flow impossible.
- High: revoke/expiry bypass, stale overwrite, unusable keyboard/AT, broken install/offline, catalog rights breach.
- Medium: recoverable state/copy/layout/observability defect.
- Low/Observation: polish 또는 future evidence.

Repository-owned Critical/High는 0이며 local fixture required 자동행은 PASS했다. Q-11/Q-25/Q-30 infra와 Q-31..Q-42/Q-46/Q-47 real-service 부분은 local capability를 허위로 낮추는 항목이 아니라 production gate를 `BLOCKED_EXTERNAL`로 유지하는 독립 증거다. 어느 행도 local fixture PASS를 실제 release PASS로 승격시키지 않는다.
