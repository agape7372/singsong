# 싱송 공간형 앱 컨셉 데모 설계

> 상태: 탐색 프로토타입 승인
> 날짜: 2026-07-22
> 범위: 기존 제품 코드를 변경하지 않는 `design-lab/space-concepts` 데모 10종

## Understanding Lock

- 싱송의 앱 본체는 티켓이 아니다. 약속을 만들고 노래를 모으는 동안 사용자는 하나의 `공간` 안에 있어야 한다.
- 티켓은 곡, 순서, 시간, 비용을 정리한 뒤 `약속 확정`을 실행했을 때 처음 등장하는 보상 artifact다.
- 플랜 단계에서는 절취선, 펀칭, 바코드, 일련번호, 발권지 비례 같은 티켓 문법을 사용하지 않는다.
- 검색, 곡 담기, 순서 조정, 인원과 예상 비용은 동일한 기능 계약으로 유지한다.
- 10개 데모는 기능 차이가 아니라 공간 은유, 정보 밀도, 브랜드 표현 방식의 차이를 비교한다.
- 제작물은 클릭 가능한 반응형 HTML이며 기존 Next.js 앱 라우트와 저장소에는 연결하지 않는다.

## 초기 디자인

모든 데모는 `입장 → 함께 고르기 → 정리하기 → 확정하기`의 공간적 흐름을 공유한다. 홈은 티켓 한 장이 아니라 방, 로비, 작업대, 갤러리, 관제 데스크처럼 여러 활동 구역이 연결된 장소로 보인다. 현재 곡 수, 참여 인원, 예상 시간과 비용은 장소의 표지판이나 작업 상태로 표현하며, 발권 장식은 사용하지 않는다.

각 데모에서 곡 추가와 인원 변경을 직접 조작할 수 있다. `약속 확정` 전에는 티켓이 화면에 존재하지 않으며, 확정 버튼을 누른 뒤 공통 Ticket Reveal 대화상자가 열린다. 이때만 절취선, 양쪽 펀칭, 번호, 발권 정보가 등장한다. 따라서 티켓은 앱의 배경 장식이 아니라 준비가 끝났다는 명확한 사건이 된다.

## 위험도

프로토타입 자체는 격리되고 되돌리기 쉬워 `Moderate`다. 다만 대표안을 실제 앱에 적용하면 정본의 Session Strip 계약을 바꾸므로 별도 제품 결정과 회귀 검증이 필요하다. 이 데모 단계는 정본 변경이나 production 적용을 승인하지 않는다.

## 수용 기준

- 10개 데모가 모바일과 데스크톱에서 모두 읽힌다.
- 앱 본체 어느 곳에도 티켓 절취선, 펀칭, 바코드가 나타나지 않는다.
- 곡 추가, 인원 증감, 약속 확정이 키보드와 포인터로 작동한다.
- 티켓은 확정 이후 대화상자에서만 나타난다.
- 각 컨셉의 공간 은유가 이름을 보지 않고도 시각적으로 구분된다.
- 텍스트 대비, 포커스 표시, 44px 터치 영역, reduced motion을 고려한다.
- 대표 후보를 고를 때 브랜드성, 사용성, 공간감, 구현 난이도를 독립적으로 비교할 수 있다.

## 구현·검수 결과

- `design-lab/space-concepts/index.html`에 10개 컨셉을 하나의 비교 가능한 디자인 랩으로 구현했다.
- 곡 추가·삭제, 2–8명 인원 변경, 시간·비용 재계산, 확정 후 티켓 발권이 작동한다.
- 계획 화면에는 티켓 절취선·펀칭·바코드가 없고 발권 대화상자에만 존재한다.
- 390px 모바일에서 문서 가로 넘침은 `0px`다.
- 10개 데모 전체의 Axe 자동 접근성 위반은 `0건`이다.
- 44px보다 작은 데모 조작 요소는 `0개`이며, 첫 키보드 포커스에는 `3px solid` outline이 표시된다.
- 상호작용 검증에서 5명·4곡 변경 후 티켓이 24–32분, 31,500–42,000원으로 갱신됐다.
- 최종 캡처와 JSON 증거는 `C:/tmp/singsong-space-concepts/`에 생성했다.

## 비교 shortlist

- **10 싱송 스테이션**: 세 계열의 균형이 가장 좋아 실제 제품 확장 후보로 우선 검토할 만하다.
- **01 모임의 방**: 약속을 함께 만드는 정서와 공간 은유가 가장 직접적이다.
- **07 컨트롤 데스크**: 입장 직전 빠른 조작과 정보 판독성이 가장 강하다.
- **04 포스터 홀**: 캠페인과 브랜드 인지도 측면에서 가장 강한 시각적 개성을 가진다.

이 shortlist는 production 채택 결정이 아니다. 실제 앱 반영 전에는 사용자 비교, 기존 기능 회귀, 정본 문서 갱신이 필요하다.
